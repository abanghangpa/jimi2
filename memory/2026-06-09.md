## 2026-06-09 - Scanner Recovery & FRED Hang Diagnosis

### 🛠️ The "Great Hang" of `scanner.py`
- **Issue**: `scripts/scanner.py` was hanging indefinitely immediately after startup.
- **Red Herring**: `fatal: bad revision 'HEAD'` error appeared in output. Diagnosis: This is a non-fatal shell/env warning and not the cause of the freeze.
- **Root Cause**: `strace` revealed the process hanging on a TCP connection to the FRED (St. Louis Fed) server (`23.199.152.194`).
- **The Trigger**: Import-time initialization in `src/modules/cascade_meta.py` and network calls in `src/utils/macro_fetch.py` were triggering blocking requests to FRED's API during the startup phase, before the main logic could even log its progress.
- **Solution**: 
    - Implemented a "nuclear" mock of all FRED network requests in `src/utils/macro_fetch.py` and `src/modules/m68_yield.py`.
    - Set a global socket timeout (`socket.setdefaulttimeout(30)`).
    - Verified recovery by running the scanner with a 300s timeout.
- **Result**: Scanner now boots and executes in < 10 seconds.
